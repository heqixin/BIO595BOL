7.2 Phylogenetic Diversity

Phylogenetic diversity can me estimted in multipe ways. Here we will focus on the the three most common metrics: Faith’s phylogenetic diversity (PD); mean phylogenetic distance (MPD); and mean nearest taxon/neighbor distance (MNTD/MNND). Calculations of phylogenetic diversity requires two things - a phylogeny and groupings of taxa. Let’s work with examples from the picante package, which compiles functions for community phylogenetic analyses. We will discuss community phylogenetics later in the course.

data(phylocom) # this will load some example data 
phylocom # what kind of object is this? what are its contents?

plot(phylocom$phylo) #tree
View(phylocom$sample) #community data matrix
View(phylocom$traits) #trait data


7.2.1 Preprocessing

The calculation of phylogenetic diversity metrics requires a community data matrix. Rows must represent groups and columns must represent different taxa. The contents of this matrix can represent presence/absences and/or abundance. Importantly, functions in picante assume that the taxon names on your phylogeny and associated data are in the SAME ORDER.

tree <- phylocom$phylo
comm <- phylocom$sample

## prune tree to only include taxa in community matrix
match(tree$tip.label, colnames(comm)) #NAs indicate non-matches
is.na(match(tree$tip.label, colnames(comm))) #are the matches NAs?
any(is.na(match(tree$tip.label, colnames(comm)))) #are any matches NAs? i.e., are there any mismatches?

tree$tip.label[which(tree$tip.label %in% colnames(comm))]
tree$tip.label[which(!tree$tip.label %in% colnames(comm))]
nonmatching <- tree$tip.label[which(!tree$tip.label %in% colnames(comm))]

prunedtree <- drop.tip(tree, nonmatching)
plot(prunedtree)

## We need to make sure the columns of the community matrix are in the same order as the tree tips
comm <- comm[, prunedtree$tip.label]
View(comm)

### OR, we can make use of a wrapper function that does all of these steps for us:
match.phylo.comm(phylocom$phylo, phylocom$sample)
treecomm <- match.phylo.comm(phylocom$phylo, phylocom$sample)
prunedtree <- treecomm$phy
comm <- treecomm$comm


7.3 Estimating phylogenetic diversity metrics

Once we have matched the different sources of data, we can proceed to estimate metrics of phylogenetic diversity. The pd function calculates Faith’s phylogenetic diversity (i.e., the sum of all branch lengths connecting a group); mpd calculates mean phylogenetic distance, and mntd calculates mean nearest taxon/neighbor distance. The latter two functions require us to first calculate all pairwise distances between taxa.

# Phylogenetic Diversity (PD)
pd(comm, prunedtree)
test.pd <- pd(comm, prunedtree)
plot(test.pd)
barplot(test.pd$PD, names.arg = rownames(test.pd))

# Mean Phylogenetic Distance (MPD)
pairwise.pd <- cophenetic(prunedtree) # calculates all pairwise phylogenetic distances among taxa
mpd(comm, pairwise.pd)
test.mpd <- mpd(comm, pairwise.pd)

# Mean Nearest Taxon Distance (MNTD/MNND)
mntd(comm, pairwise.pd)
test.mntd <- mntd(comm, pairwise.pd)


Let’s examine how different these measures of phylogenetic diversity are.

# combine our results and examine trends
cbind(test.pd, test.mpd, test.mntd)
test <- cbind(test.pd, MPD = test.mpd, MNTD = test.mntd)
plot(test$PD, test$MPD)
cor.test(test$PD, test$MPD)

# can we do this all at once?
pairs(test)
cor(test)


Phylogenetic diversity does not have to be calculated for spatially delimited communities of taxa. Taxa can be grouped temporally, by status, by functional groups, by clade, etc. Let’s try another example where our data is not quite in the necessary format. The reshape2 package contains several functions that are useful for reshaping/reformatting your data.

## First, we need to get our data into the correct format
comm.df <- t(df)
comm.df <- melt(comm.df)
comm.df2 <- comm.df[which(comm.df$Var2 == "status"), c("Var1", "value")] #subsetting data to only what we need
comm.df2 <- acast(comm.df2, value ~ Var1, fill = 0, fun.aggregate = length) #aggregating the data into a presence absence matrix

comm.df <- acast(comm.df, value~Var1, fill = 0, fun.aggregate = length, subset = .(Var2 == "status")) # we can also do this inside the acast function
comm.df <- comm.df[, tree$tip.label] # we could have also used the match.phylo.comm function at this stage

## Next, calculate phylogenetic diversity
pd(comm.df, tree)
pairwise.pd <- cophenetic(tree)
mpd(comm.df, pairwise.pd) #results are in order of comm.df rows
mntd(comm.df, pairwise.pd)  #results are in order of comm.df rows
