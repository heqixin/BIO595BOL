5.2 Examining phylogenetic trees

Phylogenetic trees can be saved in a number of different formats. The most common two formats are nexus and newick. Phylogenetic tree files can have several different types of extensions irrespective of their format - e.g., ".txt", ".phy", ".tre", ".tree", ".nwk", ".newick", ".nex", ".nexus". Regardless of the extension, these are all text files. Though using standardized and broadly recognized file extensions is recommended, this is not always the case. File extensions are just tags that help people/programs/functions guess the type of file - they do not determine what is actually in the file or how it is formatted. Hence it is often wise to examine (text) files in a text editor (e.g., Notepad, Notepad++, TextEdit, BBEdit) when you are not sure what format they are in. You can also take a look in R using the readLines function. Let’s take a look at two different tree files.

readLines("Daniel_tree.tre")

readLines("Daniel_tree.nex")


Once we have determined what format these trees are in, we can read them into R with the appropriate function.

?read.tree

?read.nexus

tree <- read.tree("Daniel_tree.tre") # reads newick files
#tree <- read.nexus("Daniel_tree.nex") # reads nexus files

# plot tree
plot(tree)
plot(tree, type = "p", use.edge.length = FALSE, show.tip.label = TRUE,
     edge.color = "blue")

Examine the plotted tree. Is the tree rooted? What appears to be the outgroup? Unless otherwise specified, a tree read into R as a class phylo object is typically rooted at the first node. However this is not always guaranteed. The find_root function in the castor package identifies the root node.

is.rooted(tree)

find_root(tree)

plot(tree)

nodelabels()


Phylogenetic tree objects in R (class phylo) are essentially lists of several types of information that together comprise a tree. Let’s take a look at them.

tree$tip.label

tree$edge

tree$Nnode

tree$edge.length

length(tree$tip.label)

tip.labels are the names of the “leaves” of the tree (e.g., species names, gene names); Nnode stores the number of nodes; and edge.length stores branch lengths. $edge returns a table where the row numbers represent the edge (i.e., branch) numbers, the first column represents the parent node and the second column the child tip/node.


We can manipulate trees by altering these elements. For instance:

# changing tip labels
tree1 <- tree
tree1$tip.label <- gsub("t", "tip", tree1$tip.label)
plot(tree1)

# pruning trees
tree2 <- drop.tip(tree, "t10")
length(tree2$tip.label)
plot(tree2)

tree3 <- drop.tip(tree, c("t1", "t5"))
length(tree3$tip.label)
plot(tree3)

# removing branch lengths
tree4 <- tree
tree4$edge.length <- NULL
plot(tree4)


5.3 Plotting phylogenetic trees

We can plot trees in several different ways. Below are some examples.

# rotating nodes
tree2 <- ape::rotate(tree, 12)
plot(tree2)

?plot.phylo()

# cladograms
plot(tree, type = "cladogram")

# phylograms
plot(tree, type = "phylogram")

# circular
plot(tree, type = "fan")

# radial
plot(tree, type = "radial")

# tidy
plot(tree, type = "tidy")

# ladderize trees
tree.l <- ladderize(rooted.tree)
plot(tree.l, use.edge.length = FALSE)

# changing the root
rooted.tree <- root(tree, "t3", resolve.root = TRUE)
plot(rooted.tree, type = "cladogram") 

rooted.tree <- root(tree, c("t2", "t6"), resolve.root = TRUE)
plot(rooted.tree, type = "cladogram", use.edge.length = FALSE) 
plot(rooted.tree, type = "phylogram", use.edge.length = FALSE) 

unrooted.tree <- unroot(tree)
plot(unrooted.tree)
