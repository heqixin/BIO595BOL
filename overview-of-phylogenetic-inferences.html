Chapter 12 Overview of Phylogenetic Inferences

12.1 MEGA

MEGA stands for Molecular Evolutionary Genetics Analysis. It is a comprehensive tool for phylogenetic inferences and hypothesis testing with a user-friendly GUI interface developed by Kumar, Tamura and Nei (1994). It has incorporated different algorithms to perform sequence alignment, model-testing, phylogenetic inference, time-tree calibration and analyses of selection. The current version, MEGA11 was extended in 2021, including new algorithms for estimating molecular clocks.

12.2 View and Align Sequences

Click DATA and open the bear_cytb.fas file, and choose align. This file contains Mitochondria Cytochrome C sequences of all bear species. Click the W symbol for ClustalW multiple sequence alignment. We can also use a different aligning algorithm called muscle. In this case, because all the sequences are full-length protein coding sequences, and there are no insertion-deletion mutations occurred in this gene, the alignment looks exactly the same as our input. We then click Translated Protein Sequences for DNA to amino acid translation. When asked for codon standard, we choose no, and then in the genetic code window, we select Vertebrate Mitochondrial.

We save the session as a mega session file .mas. When we reopen the session file and click analyze, we will see a Sequence Data Explorer. The data explorer gives a rich information regarding phylogenetic/parsimony informative sites, degenerate sites, etc. Phylogenetic inferences can only be performed if the sequences have enough number of informative sites, i.e., sites that have at least two or more tips that are different from the rest of the sequences. Singletons cannot be used to infer relationships.

Let’s align a different sequence file mammal_mit.fas, in which the sequences are composed of coding and non-coding regions. After alignment, we can see that there are regions well-aligned, and regions with a lot of gaps. These regions usually indicate a low-confidence alignment, and we do not want to include them in the final phylogenetic inference. We can remove these regions before inferring phylogenetic trees.

12.3 DNA/protein evolution model

Next we need to select the most appropriate DNA evolution model for the analyzed sequences. Let’s click MODELS->find best DNA/protein models. After DNA model testing, we choose the most appropriate DNA evolution model for phylogenetic inference. In this case, the most appropriate one is “HKY+G”, as it has the lowest BIC values.

12.4 Phylogenetic inference

Under phylogeny, we can select neighbor-joining tree. In the previous example, the sequence evolution model with the highest support is HKY, HKY is essentially K2P with uneven base frequencies. Let’s select K2P and Gamma distribution of the rates. We select Bootstrap method as the phylogeny testing.

In the tree explorer, we can set the node to display “statistics info”. In this case, it displays the percentage bootstrap tree that supports each of the node.

12.5 Estimate divergence times

Click Clocks-> Compute timetree, and click RelTime-ML, choose sequence data, tree file and specify outgroup. In the choose analysis settings, choose the right DNA evolution model.

12.6 Exercises

  1. Describe the differences among sites that are parsimony informative sites, singletons, 0-fold, 2-fold, 4-fold degenerate sites.

  2. What’s the percentage of informative sites contained in the cyt-b sequences in our example?

  3. Under phylogeny inference, build an “ML” tree using HKY+G model, and compare with the “NJ” tree we built earlier. Are they different from each other? What about the bootstrap values?

  4. In timetree wizard, use the information in bear_taxa.tsv and constrain nodes use two sets of fossil information. Compare the estimated divergence times from the two trees and describe the differences in the time estimates.

Example: - click the node you would like to put an constraint on. For example, we would like to constrain the node of Ursus americanus-Ursus thibetanus-split.

  • After clicking the node, we click the clock-plus symbol. From the bears_taxa.tsv, we know that the earliest record of Ursus americanus fossil is 1.84 million years ago, while Ursus thibetanus’s record is 1.18 million years ago. Therefore, the minimum age the split between these two species is 1.84 mya. We therefore choose Min Time Only for the constraint and insert the number.

  • Parictis montanus belongs to the oldest known genus of bear. We therefore constrain the maximum of the crown age of all ursinae (excluding Ailuropoda melanoleuca[Giant Panda]) to be 37.2.

We can then obtain a calibrated timetree.
