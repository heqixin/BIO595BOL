3.1 What is programming?

Attempting to get a computer to complete a specific task without making mistakes. Computers only understand machine code, consisting of 0s and 1s. Programming languages are bridges between human and machine languages. It translates a format of writing with a specific syntax to machine code. It’s far more simple than human language as long as you master the syntax, and get used to the logic of organizing it. Remember, computers are dumb - they can only do exactly what you tell them to do so you need to be specific, logical, and consistent in your programming.


3.2 R

R is a language and environment for statistical computing and graphics, but can be used for much more. It comes with many built-in functions, graphical capabilities, and third-party packages.


3.3 Installation

RStudio is an integrated development environment (IDE) that works the best with R, but it can also be used to test python or C or bash code. RStudio is just an interface that makes it a bit more convenient to interact with R - in other words you are programming in R, not RStudio.

3.4 Variables

A variable provides us with named storage that our programs can manipulate. A variable in R can be stored as an atomic vector, group of atomic vectors or a combination of many R objects. The variables can be assigned values using leftward, rightward and equal to operator.

x <- 1
x

1 -> x
x

x = 1
x

We can also assign a vector to a variable:

x <- c(5, 8, 14, 91, 3, 36, 14, 30)
x

And do simple arithmetic:

x + 2

Note that R is case sensitive - if you ask for X instead of x, you will get an error.

X
Error: object 'X' not found

3.4.2 Variable naming conventions

Naming your variables is your business, but there are 5 conventions (http://www.r-bloggers.com/consistent-naming-conventions-in-r/) to be aware of: A valid variable name consists of letters, numbers and the dot or underline characters. The variable name starts with a letter or the dot not followed by a number.

  • alllowercase: e.g. adjustcolor
  • period.separated: e.g. plot.new
  • underscore_separated: e.g. numeric_version
  • lowerCamelCase: e.g. addTaskCallback
  • UpperCamelCase: e.g. SignatureMethod

3.5 Subsetting data

You can subset vectors using x[ ] notation. Square brackets allow you to look inside objects.

x

x[5]

You can use a : to quickly generate a sequence:

1:5

and use that to subset as well:

x[1:5]


3.6 Logical values

R also has standard conditional tests to generate TRUE or FALSE values (which also behave as 0s and 1s). These are often useful for filtering data (e.g., identifying all values greater than 5). The logical operators are <, <=, >, >=, == for exact matches, and != for non-matches.

  x3 <- c(5, 8, NA, 91, 3, NA, 14, 30, 100)

  x3 > 75
 
  x3 == 40
 
  x3 > 15

And you can perform operations on those results:

sum(x3 > 15, na.rm = T)

or save the results as variables:

result <-  x3 > 3
result

Think about what we just did here - what does the resulting output mean? How did we get this value?

3.6.1 Missing: dealing with NA values

Missing values can take many forms. The most common one you will encounter is NA.

is.na(x3)

x3[!is.na(x3)]

mean(x3)

mean(x3, na.rm = T)


3.7 Generating Data

There are many ways to generate data in R. Above, we manually defined a vector of numbers, x. We can also use functions and formulas. For instance:

to generate a sequence of numbers:

seq(from = 0, to = 1, by = 0.25)

to repeat elements:

rep(1:3, 3)
rep(1:3, each = 3)

to generate random numbers that follow a normal distribution:

a <- rnorm(100, mean = 0, sd = 10)

Let’s visualize those values in a histogram:

hist(a)

We’ll cover much more sophisticated graphics later in the course.


3.8 Data Structures and Types

R has a wide variety of data types including vectors (numerical, character, logical), factors, matrices, data frames, and lists.

3.8.1 Vectors

Vectors are a basic data structure in R, and are arrays of elements of the same type. Atomic vectors can be logical, integer, double (numeric), character, complex (e.g., imaginary numbers) or raw (bytes).

a <- c(1, 2, 5.3, 6, -2,4) # numeric vector
b <- c("one", "two", "three") # character vector
c <- c(TRUE, TRUE, TRUE, FALSE, TRUE, FALSE) #logical vector
d <- 1:10

The functions typeof(), length(), class() and str() provide useful information about your vectors and R objects in general.

Examine the data type of vector a:

typeof(a)

Double vectors are also known as numeric vectors in R. This includes decimal values with precision. In R, numbers are double by default.

Examine the type and length of vector b:

typeof(b)

length(b)

Character vectors are the most complex type of atomic vector because each element of a character vector is a string and a string can contain an arbitrary amount of data. Strings in R can contain the alphanumeric characters and symbols. The easiest way to denote that a value is a character in R is to wrap the value inside single or double inverted commas.

Examine the data type of vector c:

typeof(c)

Logical vectors are the simplest type of atomic vector as they take only three possible values: FALSE, TRUE, and NA. Logical vectors can be constructed with comparison operators.

Examine the data type of vector d:

typeof(d)

Integers can only contain whole numbers. They are functionally interchangeable with double (numeric) vectors for most purposes.

Examine the data structure of vector a:

str(a)

You can convert vectors from one to another type

as.character(a)

as.character(c)

as.logical(a)

as.numeric(b)

as.numeric(c)

You can also use matrices (2-dimensional arrays of numbers):

y <- matrix(1:9, ncol = 3)
y

Matrices behave much like vectors:

y + 2

and have 2-dimensional indexing:

y[2, 3]

3.10 Data Frames

Dataframes are similar to matrices, but more flexible. Matrices must be all the same type (e.g. all numbers), while a dataframe can include multiple data types (e.g. text, factors, numbers). Dataframes are commonly used when processing and analyzing data in R.

data <- data.frame( x = c(11, 12, 14),
                    y = c("a", "b", "b"),
                    z = c(T, F, T))
data

You can view dataframes and matrices in a spreadsheet-like format using the View function. You to interact with the spreadsheet in RStudio (e.g., search, filter, order), though changes will not be saved.

View(data)

You can extract values from datatframes several ways. Columns and rows can be referred to by their names and/or number (order).

# columns
data$y

data[["y"]]

data[, 2]

data[[2]]

colnames(data)

# rows
data[3, ]

data["3", ]

rownames(data)

# specific cells
data[1, 2]

data[1, "y"]

data["1", 2]

data["1", "y"]

We can apply functions to different subsets of a dataframe:

mean(data$x)

mean(data[["x"]])

mean(data[, 1])

mean(data[[1]])

Some common operations that can be done on dataframes:

# save subset to new object
data2 <- data[1:2, c("x", "y")]
View(data2)

# reversed values
rev(data$x)

# sort data
sort(data$x, decreasing = TRUE)
sort(data$y, decreasing = TRUE)
sort(data$z, decreasing = FALSE)

# compute the frequency of each value
table(data$y)

# extract unique values
unique(data$z)

Add new variables (columns):

data$w <- data$x + 5
data$v <- c("excellent","good","fair")
