8.2 Definition of Community Phylogenetics

The species composition of a biological community results from the interplay between evolutionary and ecological processes (Ricklefs 1987) and thus communities of organisms can exhibit non-random patterns of evolutionary relatedness among co-occurring species (reviewed in Webb et al. 2002). Community phylogenetics (i.e., phylogenetic community ecology) seeks to understand the processes that govern species assemblages (i.e., communities) on the basis of the phylogenetic relationships shared among co-existing taxa. We will only cover phylogenetic approaches in the lab, but see https://rpubs.com/gksmith/11473 for an excellent tutorial on related taxonomic approaches to questions of community ecology in R.


8.2.1 Alpha diversity

Alpha diversity is a term used to describe within-sample diversity. The phylogenetic diversity calculations we conducted in the previous lab session were all estimations of alpha diversity (i.e., the diversity of a group). Let’s revisit what we did.

plot(phylocom$phylo) #tree
View(phylocom$sample) #community data matrix
View(phylocom$traits) #trait data

## we need to make sure the columns of the community matrix are in the same order as the tree tips and comprise the same taxa
match.phylo.comm(phylocom$phylo, phylocom$sample)
treecomm <- match.phylo.comm(phylocom$phylo, phylocom$sample)
prunedtree <- treecomm$phy
comm <- treecomm$comm

# Phylogenetic Diversity (PD)
pd(comm, prunedtree, include.root = FALSE)
alpha.pd <- pd(comm, prunedtree)

# Mean Phylogenetic Distance (MPD)
pairwise.pd <- cophenetic(prunedtree) # calculates all pairwise phylogenetic distances among taxa
mpd(comm, pairwise.pd)
alpha.mpd <- mpd(comm, pairwise.pd)

# Mean Nearest Taxon Distance (MNTD/MNND)
mntd(comm, pairwise.pd)
alpha.mntd <- mntd(comm, pairwise.pd)

# combine results
alpha.diversity <- cbind(alpha.pd, MPD = alpha.mpd, MNTD = alpha.mntd)


Let’s look at how these metrics of alpha diversity are calculated. One way to do this is to type the function name into the console (e.g., “pd”) - in many cases this will display the R code than makes up the function. You can go through this code line by line to understand what is going on under the hood, and recreate/modify it yourself.

#pd
pd(comm, prunedtree, include.root = F)
clump1 <- colnames(comm)[which(comm[1, ] > 0)]
clump1tree <- drop.tip(prunedtree, prunedtree$tip.label[which(!prunedtree$tip.label %in% clump1)])
sum(clump1tree$edge.length)

8.2.2 Beta diversity

Sometimes referred to as ‘turnover’, beta diversity is a comparison among groups of taxa; i.e., between-sample diversity. It is generally measured as the amount of compositional change between groups of taxa. In the context of community phylogenetics, beta diversity is a measure of the phylogenetic similarity or dissimilarity of two communities.

comdist(comm, pairwise.pd) #beta MPD

beta.MPD <- comdist(comm, pairwise.pd)

MPD.dendrogram <- hclust(beta.MPD)

plot(MPD.dendrogram)

comdistnt(comm, pairwise.pd) #beta MNTD

beta.MNTD <- comdistnt(comm, pairwise.pd)

8.3 Standardized effect sizes

Standardized effect size statistics remove the units of the variables in the effect. They can help you evaluate how large or small an effect is when the units of measurement aren’t intuitive, and compare results across studies/samples. We can calculate the standard effect sizes (SES) of phylogenetic diversity metrics by dividing the difference between the observed value and values for randomly generated (null) communities by the standard deviation of the randomly generated values.

(meanobs - meanrandom) / sdrandom

This also allows us to test whether the observed values are significantly different from random/null expectations. Null models implemented in the package are as follows:

taxa.labels Shuffle distance matrix labels (across all taxa included in distance matrix)
richness Randomize community data matrix abundances within samples (maintains sample species richness)
frequency Randomize community data matrix abundances within species (maintains species occurence frequency)
sample.pool Randomize community data matrix by drawing species from pool of species occurring in at least one community (sample pool) with equal probability
phylogeny.pool Randomize community data matrix by drawing species from pool of species occurring in the distance matrix (phylogeny pool) with equal probability
independentswap Randomize community data matrix with the independent swap algorithm (Gotelli 2000) maintaining species occurrence frequency and sample species richness
trialswap Randomize community data matrix with the trial-swap algorithm (Miklos & Podani 2004) maintaining species occurrence frequency and sample species richness

Also see the Phylocom manual for more information.

pd(comm, prunedtree) #plain old phylogenetic diversity
ses.pd(comm, prunedtree, runs = 99) #for real analyses use AT LEAST 999 runs, or more like 9999
ses.pd(comm, prunedtree, runs = 99, null.model = "taxa.labels") #you can try different null models (see ?ses.pd for more info)

mpd(comm, pairwise.pd)
ses.mpd(comm, pairwise.pd, runs = 99) #does not incorporates abundance data

mpd(comm, pairwise.pd, abundance.weighted = TRUE)
ses.mpd(comm, pairwise.pd, runs = 99, abundance.weighted = TRUE) #this incorporates abundance data

mntd(comm, pairwise.pd)
ses.mntd(comm, pairwise.pd, runs = 99, abundance.weighted = FALSE)

Positive SES values indicate that the observed value is higher than null expectations (obs.z) Negative SES values indicate that the observed value is lower than null expectations (obs.z) p-values can be used to assess significance of these deviations (obs.p)

We can also calculate SES for beta diversity metrics.

ses.mpd(comm, pairwise.pd, runs = 99, abundance.weighted = FALSE)

ses.mntd(comm, pairwise.pd, runs = 99, abundance.weighted = FALSE)
