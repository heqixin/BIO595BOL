2.1 What is Unix and why do we use it?

Unix is an operating system, which is the software that lets you interface with the computer. Apple’s OS, Linux are among the popular variants of Unix used today. In this lab, we’ll focus the use of Unix shell, the interface that is used to communicate with the core of the operating system. Using the shell is quite a different experience than the usual graphical user interfaces (GUIs). Within the shell, we directly type commands to ask certain tasks to be done, instead of interacting with buttons and windows.

Learning basic command lines is very important for biologists, especially researchers because many small programs are available or can be developed to perform simple tasks, scaling up the analyses on batches of files, and integrate different programs into a analyzing pipeline. Such scripts make your work 100% reproducible, easy to debug, and avoid random mistakes introduced because you clicked wrong buttons on a GUI interface.

Lastly, biological research replies heavily on large text files of genetic and/or character sequences. Unix is designed around text viewing and editing. Phylogenetic analyses usually involves large amount of calculation, and requires high-performance computing platforms, which are unix-based. Mastering the skills of command-line operations will allow you to work with large files effortlessly.

When a new shell starts, it always opens in your home directory. The tilde (~) symbol is shorthand for the home directory in unix. The exact path to the home directory varies among different operating systems. To see its actual location, type:

In OS X, you should see /Users/<username>. In linux, you should see /home/<username>. In windows, it would be /c/Users/<username>. When specifying a path, if it doesn’t start with “/”, then it refers to subfolders or files under the current directory. If the path starts with “/”, it denotes directories from the topmost “root” directory.

Next, let’s explore the bsic commands to navigate the hierarchical directory system.

pwd Print the path of the current working directory. This command prints your current location of the shell.

2.3.3 Absolute and Relative directory

One can navigate through the directory hierarchy by providing either the absolute path (i.e., starts with “/”) or a relative path (i.e., utilize “..”, “-”, “./”, or directly type the subdirectory name in the current folder).

# absolute path
cd ~/lab1_unix
# see all the subdirectories and files in this directory
ls

Tip1: The Tab key provides options or autocomplete when you only remember part of the file/folder name. For example, after typing cd u, and press the Tab key, if the current directory only has one file/folder that starts with “u” (e.g., unix), then the command will autocomplete the command to be cd unix. If there are multiple ones that starts with “u”, the list of options will be shown, and you can keep typing the following letter until there’s only one unique option and press Tab again. This is very helpful when folder or file names are long or confusing.

# relatvie path
#cd lab1_unix
cd ./data
cd ../sandbox
cd -
# ./ or . refers to the current directory
cd ./
pwd
ls
# try the Tab key to test autocomplete.
cd ../../

Tip2: space in unix command indicates separation of arguments. Thus, spaces within a directory or file name becomes confusing. Try avoid using spaces in naming directories or files. When existing files/directories contain spaces, you can refer to it with quotation marks or a backslash (\) before a space:

# if we would like to open a directory
# called "unix names should not contain spaces"

cd "unix names should not contain spaces"

2.9 Useful wildcards

Wildcards are special symbols that work as placeholders for one or more characters.

* stands for any characters, length from 0 to more zero or more, with the exception of a leading dot. Unix uses a leading dot for hidden files, so this means that hidden files are ignored in a search using this wildcard (show hidden files using ls -a).

? is a placeholder for any single character, again with the exception of a leading dot.

[] The square brackets match any one of characters inside the brackets. For example, [xyz] will match any of “x”, “y”, or “z”. A range of characters in the alphabet can be matched using the hyphen. For example, [a-z] will match all the lower case alphabets, [a-zA-Z] will match all the alphabets, [0-9] will match any single numbers.

# change into the directory
cd lab1_unix/data/miRNA
# count the numbers of lines in all the .fasta files
wc -l *.fasta
# print the first two lines of each file
# whose name starts with mi
head -n 2 mi*
# print the last three lines of each file
# whose name starts with p or s
tail -n 3 [ps]*.fasta

2.11 Search and select lines using grep

grep <pattern> <filename> will try to match all the lines in the file and output the lines with a match to the pattern specified. This is very useful when we would like to pick out sequences of certain species or other specifications from a large sequence file.

# change into the directory
cd lab1_unix/data
# inspect Marra2014_data.fast
less Marra2014_data.fasta
q
# count how many sequences there are for the gene isogroup00001
grep isogroup00001 Marra2014_data.fasta | wc -l
# search for gene isogroup00001 and show the first two
grep isogroup00001 Marra2014_data.fasta | head -n 2
# alternatively, with "-c" option, it outputs the count of lines directly
grep -c isogroup00001 Marra2014_data.fasta
# "-w" option matches whole words only, compare these two commands
grep -c -w isogroup000 Marra2014_data.fasta
grep -c -w isogroup00001 Marra2014_data.fasta
# "-i" ignores case
grep -c -i ISOgroup00001 Marra2014_data.fasta
grep -c ISOgroup00001 Marra2014_data.fasta
# use wildcards within grep, we need to add "\" before them
grep -c isogroup0000\[1-9] Marra2014_data.fasta
grep -c isogroup\* Marra2014_data.fasta
# To print all the lines that do not match a given pattern, use the option -v.
grep -c -v isogroup\* Marra2014_data.fasta
# grep can also search through many files at the same time
grep -c contig *.fasta

2.12 Substituting characters using tr

tr <original characters> <substitution> will fine all the occurrences original characters and substitute with the second. If no specification, the result of substition will print out on the screen. You can redirect it to a new file.

#Substitute all characters a with b
echo "aaaabbb" | tr "a" "b"
#Substitute every digit in the range 1 through 5 with 0:
echo "123456789" | tr 1-5 0
# special: translate all lower case letters to upper case
echo "ACtGGcAaTT" | tr [:lower:] [:upper:]
# Delete all occurrences of a:
echo "aaaaabbbb" | tr -d a
# substitute delimiters:
echo "species1 species2" | tr " " ","
# substitute space with tab
echo "species1 species2" | tr " " "\t"

2.13 cut, sort and unique

sort Sort the lines of a file and print the result to the screen. By default, it sorts the lines using alphabetical order. Use option -n for numerical sorting and -r to reverse the order. The option -k is useful to sort a delimiter-separated (specified by -t) file by a specific column

cut selects a specific column separated by a delimiter (usually ends with csv). -d specifies the delimiter, e.g., “,”, “;”, or ” “,”. -f specifies the column number or range, e.g., -f 1 or -f 2-4 selects the first or second to the fourth columns of a file.

# go to the sandbox folder
cd lab1_unix/sandbox
less test.csv
q
# it's an ";" delimited file of characteristics of mammal species
# sort the header by "AdultBodyMass_g"
head -n 1 test.csv
# AdultBodyMass_g is the 6th column, let's confirm first
head -n 1 test.csv| cut -d ";" -f 6
# get species name and Adult body mass, and sort by body mass
cut -d ";" -f 5,6 test.csv| sort -n -t ";" -k 2
# get unique list of genus number, remember to remove the header line
tail -n +2 test.csv | cut -d ";" -f 4 | sort |uniq
# get unique list of genus number, and count the total number
tail -n +2 test.csv | cut -d ";" -f 4 | sort |uniq|wc -l
